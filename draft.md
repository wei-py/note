# 前端性能优化
### 浏览器
- 合并文件，使用雷碧图，使用 base64 传输，减少 http 请求
- 通过 DNS 缓存来减少 DNS 查询
- 通过设置缓存策略，对常用不变的资源进行缓存
- 使用延迟加载的方式，来减少首屏加载是需要请求的资源。延迟加载的资源当用户
- 通过用户行为，对某些资源进行预加载，来提高用户的访问速度
- 事件代理，减少内存占用和事件注册，实现动态绑定事件
### 服务器
- 使用 cdn，来提高用户对资源请求是的速度
- 服务器使用压缩 gzip deflate 等方式对传输资源进行压缩
- 尽可能减少 cookie 的大小，将静态资源分配到其他域名下，请求静态资源避免携带不必要的 cookie
### 代码
- 把样式放在页面的 head 标签中，减少首页渲染的时间
- 避免使用 @import
- js 脚本使用 defer async 或者放在页面底部，避免脚本加载和执行阻塞页面的渲染
- 通过 js 和 css 文件进行压缩
